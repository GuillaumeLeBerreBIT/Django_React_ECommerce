Explain concepts before showing code — Guillaume asks "how does X work" before asking to implement it.

**Why:** User is learning, not just building. Understanding the why matters as much as the what.

**How to apply:**
- Use ASCII diagrams and flow charts to show architecture
- Use comparison tables (e.g. SPA vs traditional, .js vs .jsx)
- Answer the conceptual question fully, then show the code example
- When summarizing for NOTES.md, include: what was built, what each piece does, and key concepts — not just code dumps
